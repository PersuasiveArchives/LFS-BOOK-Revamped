iii. LFS Target Architectures

LFS primarily targets AMD/Intel x86 (32-bit) and x86_64 (64-bit) CPUs. The instructions also work, with some modifications, for PowerPC and ARM CPUs. To build a system for these alternative CPUs, you'll need an existing Linux system, such as an earlier LFS installation or distributions like Ubuntu, Red Hat, Fedora, or SuSE. A 32-bit distribution can also be used as a host system on a 64-bit AMD/Intel computer.

The performance gain from building on a 64-bit system over a 32-bit one is minimal. For instance, in a test build of LFS-9.1 on a Core i7-4790 system, using 4 cores, the results were: 

Architecture Build Time     Build Size
32-bit       239.9 minutes  3.6 GB
64-bit       233.2 minutes  4.4 GB

As seen, the 64-bit build is just 3% faster and 22% larger than the 32-bit version. If you plan to use LFS as a server or firewall, a 32-bit CPU may suffice. However, for a desktop, where some packages need more than 4 GB of RAM, a 64-bit system is recommended.

By default, LFS creates a pure 64-bit system, supporting only 64-bit executables. Building a multi-lib system (32-bit and 64-bit support) requires compiling applications twice. This is not supported by LFS because it would complicate the educational goal of keeping the instructions minimal. However, some editors maintain a multilib fork of LFS, available at this link. Note that this is an advanced topic.
