vii. Typography

To make it easier for you to follow along, we've got some typographical conventions throughout the book. This section will give you some examples so you can see how things are formatted in Linux From Scratch. 

./configure --prefix=/usr

This is the kind of text that you'll need to type exactly as you see it. It's used for commands, unless we say otherwise. This will also be used in explanations to show which commands we're talking about.

Sometimes, we break a longer command into multiple lines. If you see a backslash at the end of a line, that means the command continues onto the next line. 

CC="gcc -B/usr/bin/" ../binutils-2.18/configure \
  --prefix=/tools --disable-nls --disable-werror

A quick tip: the backslash needs to be followed immediately by a return. Don't add spaces or tabs, as that could mess things up. 

install-info: unknown option '--dir-file=/mnt/lfs/usr/info/dir'

This fixed-width text shows the output you’d typically get on the screen after running a command. It's also how we display filenames, like /etc/ld.so.conf.

Note

If possible, configure your browser to display fixed-width text using a good monospaced font. This way, you’ll easily distinguish between characters like Il1 or O0.

Emphasis

We use this style to highlight important points or items that need extra attention.

Here’s a hyperlink for you: Linux From Scratch website

We use this format for links, both internal within the LFS community and to external sites. For example, it might be used for HOWTOs, download locations, or websites. 

cat > $LFS/etc/group << "EOF"
root:x:0:
bin:x:1:
......
EOF

This format is for creating configuration files. The first command tells the system to create the file $LFS/etc/group from whatever you type next, until it hits the "EOF" line. Basically, everything you see in this section is what you need to type.

<REPLACED TEXT>

This is used to show text that shouldn't be typed exactly as it appears, or for copy-pasting.

[OPTIONAL TEXT]

This indicates text that is optional.

Here’s an example of a man page reference: passwd(5)

This format refers to a specific man page. The number in parentheses points to a section in the manual. For instance, passwd has two man pages. As specified by the LFS installation instructions, these pages are located at /usr/share/man/man1/passwd.1 and /usr/share/man/man5/passwd.5. If the book mentions passwd(5), it's specifically referring to /usr/share/man/man5/passwd.5. Running man passwd will print the first man page it finds, usually /usr/share/man/man1/passwd.1, but to see the specific page mentioned in the book, you’ll need to run man 5 passwd.
